Hoisting in Javascript

Introduction

Hoisting is a magical thing in Javascript. Having a thorough understanding of hoisting under the hood gives us a edge on understanding the code and debugging.A lots of developers do not know what happens underneath during hoisting. Let's see how hoisting works.

Let us take a look on what are hoisted in Javascript. The following are hoisted in Javascript

  • var
  • functions
  • let
  • const

Variable "var"

Javascript var a = 10;

We could declare a variable in Javascript using the keyword var.

What do you think happens if we console log the variable "a" before it's declaration.

Javascript console.log(a);
var a = 10;

Unlike other programming languages like Java,C++, etc., the Javascript doesn't throw an error.

Console undefined

Here it says that the value of "a" is "undefined".The Javascript engine knows that there exist a variable "a" but currently the value for "a" is not initialized hence it is "undefined"

You might ask that "a" has been assigned " 10" but why console shows "undefined" for "a".Look the below snippet to understand the magic of hoisting

Javascript var a;
console.log(a);
a = 10;

What happens is before the execution of the code, the Javascript engine reads the code line by line and brings all the declarations of variables to the top/ starting part of the code. Hence when we console log the variable before it's initialization we get value as "undefined".

Function

Javascript console.log(add(10,5));
function add(a,b){
return a+b;
}

Hoisting for the functions happens in similar way as the "var" are hoisted. The Javascript engine reads the code line by line and brings all the function declarations to the top.

Javascript function add(a,b){
return a+b;
}
console.log(add(10,5));
Console 15

Variables "let" and "const"

Javascript console.log(a,pi);
let a = "hello";
const pi = 3.14; Console Cannot access 'a' before initialization

Variables "let" and "const" are hoisted in Javascript but as you expect , it does not produce "undefined" in console as it did for "var" variable hoisting.

The reason is though "let" and "const" are hoisted but they are not initialized with value "undefined".

Accessing values before initialization results in reference error.These variables are said to be in Temporal Dead Zone. It's the zone that exist from the hoisting of "let" and "const" till the initialization of "let" and "const" variables.

We could only access the values of "let" and "const" only when their line of initialization has been excuted.

Javascript let a = "hello";
const pi = 3.14;
console.log(a,pi); Console "hello" 3.14

Conclusion

We have seen with detailed examples on how hoisting happens for var, functions, let and const.And what happens we access let and const in their temporal dead zone.
